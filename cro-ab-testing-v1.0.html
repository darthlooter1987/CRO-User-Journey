What kind of experiment?

Choose how you'd like to test your Awareness Landing Page.

⚖️
A/B Testing
Test two versions head-to-head
Create a single variation of your page (different headline, hero image, or CTA) and split traffic between the original and the new version. Best when you want a clear, simple answer: does Version B beat Version A?
Best for:
Headlines Hero Images CTA Buttons Pricing
How it works:
1 You create one variation
2 We split traffic 50/50 (then auto-optimize)
3 The winner is deployed automatically
🧪
Multivariate Testing (MVT)
Test multiple elements at once
Upload different versions of multiple page elements (headline + image + button) and we'll test every combination. Best when you want to find the winning recipe AND know which ingredient matters most.
Best for:
Full Page Redesigns Multiple Elements Finding Synergies
How it works:
1 You upload assets into element buckets
2 We generate & test all combinations
3 Losers are pruned, the best recipe wins

When should the test end?

Unlike old-school testing with fixed sample sizes, our Bayesian engine watches your data in real-time and tells you when it's confident enough to call a winner.

Confidence Threshold

We run 10,000 simulations comparing Version A and Version B. When B wins in 9,500+ simulations (at 95%), we call it.

Set it and forget it

The safety net. Even before hitting your confidence threshold, the test can end early if the risk of choosing wrong drops below your comfort level.

Risk Threshold (Expected Loss)
Expected Loss Explained

Expected Loss answers: "If Version B is actually worse, how much conversion would I lose by switching?" When that number drops to near-zero, there's no reason to keep testing.
