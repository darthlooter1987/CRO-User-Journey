Product Overview

Revmatics CRO is a conversion rate optimization tool designed specifically for Shopify merchants. It analyzes a merchant's storefront, extracts brand intelligence, and automatically generates four fit-for-purpose landing pages corresponding to the purchase funnel stages: Awareness, Differentiation, Conversion, and Loyalty. The tool educates merchants on why different landing pages are needed for different campaign objectives, guides them through a personalized setup experience with a persistent global progress bar, and provides a dashboard to view, finalize, and publish generated pages. Features include responsive editing and subscription-based publishing. A/B testing is planned for a future release.

Primary User Story
As a small business owner with a Shopify account, I need to be educated on marketing activities in order for me to see the value in using the CRO tool. I need to understand what the stages of a purchase funnel are and why I am using different landing pages to facilitate different outcomes in different campaigns.
Product Flow Overview
  1. URL Entry & Landing Page Education: Merchant enters Shopify store URL and learns about funnel stages
  2. Automated Site Analysis: Tool extracts brand data (colors, fonts, logo, products) with progress animation
  3. Brand Intelligence Review: Merchant confirms extracted brand details match their identity
  4. Funnel Stage Education: Merchant learns about four funnel stages (Awareness, Differentiation, Conversion, Loyalty) as pages are generated
  5. Dashboard Carousel Showcase: Merchant views all four generated pages in carousel with readiness scores
  6. Guided Tour: Auto-starting tour walks merchant through each generated page
  7. Page Readiness Checklists: Merchant sees what's complete and incomplete on each page
  8. Page Editor & Responsive Preview: Merchant customizes pages and previews on mobile
  9. Subscription & Publish: Merchant chooses plan tier and publishes pages live
  10. Global Progress Bar: Persistent step-by-step progress bar across the top of every screen showing completion percentage
  11. A/B Testing Suite (Coming Soon): Future feature — merchant will create, run, and analyze A/B tests to optimize pages
User Stories & Acceptance Criteria
US-01
URL Entry & Landing Page Education
As a merchant, I want to enter my store URL so the tool can analyze my site and generate landing pages.
Acceptance Criteria
  • URL input field is clearly visible with placeholder text
  • CTA button ("Generate Landing Pages" or similar) triggers site extraction
  • Four value propositions are displayed on the page
  • "Four Moments" section educates user on funnel stages
  • "How It Works" steps section shows the process flow
  • Layout is clean and centered with adequate whitespace
US-02
Automated Site Analysis
As a merchant, I want my site to be analyzed automatically so I don't have to manually input brand details.
Acceptance Criteria
  • Animated progress bar displays with 5 sequential extraction steps
  • Each step shows a clear active state then transitions to completed state
  • Merchant's URL is displayed during the extraction process
  • "Next" button appears when extraction is nearing completion
  • Extraction process completes within 30 seconds
  • Loading indicators and animations maintain user engagement
US-03
Brand Intelligence Review
As a merchant, I want to review the extracted brand data so I can confirm it matches my actual brand identity.
Acceptance Criteria
  • Color swatches are displayed with accompanying hex color codes
  • Font family is detected and previewed with actual text samples
  • Logo is displayed in a brand-colored box with proper spacing
  • Product tags are listed as visual pills or badges
  • "Looks Good" action button allows merchant to proceed
  • "Regenerate" action button allows merchant to extract again
  • All extracted data is presented in an organized, scannable layout
US-04
Funnel Stage Education
As a merchant, I want to learn about purchase funnel stages so I understand why different pages exist.
Acceptance Criteria
  • Four funnel stages are presented in sequence: Awareness → Differentiation → Conversion → Loyalty
  • Each stage animates in with an icon, description, and campaign type tags
  • Global top progress bar remains visible, showing "Build Pages" as the active step
  • Page generation happens simultaneously in the background (60-second timeline)
  • "See my Landing Pages" CTA appears after all stages have animated in (also shows early after 5s for skip)
  • Educational content clearly explains the purpose of each stage and recommended campaign types
US-05
Dashboard Carousel Showcase
As a merchant, I want to see all four generated pages in an overview so I can assess them at a glance.
Acceptance Criteria
  • Four portrait cards are displayed in a horizontal carousel layout
  • Each card includes a realistic mock-up image preview of the landing page
  • Readiness score badge displays (e.g., "8/10") with color-coded progress bar
  • Stage badge and title are shown prominently on each card
  • Brief description explains the purpose of each page
  • Action buttons change based on page status: "Publish Now" for complete pages, "Finalize Now" for incomplete
  • Carousel navigation arrows and dot indicators are visible and functional
  • Cards animate in with staggered reveal on initial load
  • Hover tooltips explain the purpose of each page stage
US-06
Guided Tour
As a merchant, I want to be walked through my generated pages so I understand what's ready and what needs work.
Acceptance Criteria
  • Four-step tour auto-starts after carousel cards have fully revealed
  • Lightweight transparent overlay keeps the page visible while highlighting the active card
  • Tooltip displays step counter, title, body text, and action items
  • "Next" button advances to the next step
  • "Skip" option allows merchant to dismiss the tour at any time
  • Tour ends with a dismissable completion overlay
  • Tour can be restarted from a help icon or menu
US-07
Page Readiness Checklists
As a merchant, I want to see what's missing on each page so I know exactly what to complete.
Acceptance Criteria
  • Four checklist panels are displayed, one per landing page
  • Each checklist item shows its completion status (done or pending)
  • "Add" button allows merchant to add missing elements to a page
  • "View" button navigates to the page editor
  • Readiness percentage is shown in the panel header
  • Completed items are visually distinct (checkmark, different color, strikethrough)
  • Clicking items links directly to the relevant section in the page editor
US-08
Page Editor — Visual Preview
As a merchant, I want to preview and edit my landing page so I can customize it before publishing.
Acceptance Criteria
  • Full page frame displays a live content preview of the landing page
  • Desktop and mobile device toggle buttons are visible in the toolbar
  • Back button returns merchant to the dashboard
  • Page title and stage badge are prominently displayed in the toolbar
  • Right sidebar provides contextual hints and suggestions
  • Preview updates in real-time as edits are made
  • Page hierarchy and content structure are clear and readable
US-09
Page Editor — Edit Panel
As a merchant, I want access to editing tools so I can modify content, design, and settings.
Acceptance Criteria
  • Slide-out panel contains categorized editing tools
  • Categories include: Content, Design, Page Settings, Advanced, AI Tools
  • Each category lists specific actions (e.g., "Edit Hero Headline", "Change Colors")
  • "New" badges highlight recently added AI-powered features
  • "Soon" badges indicate upcoming features not yet available
  • Panel toggle button in toolbar shows/hides the edit panel
  • Tools are organized logically and clearly labeled
  • Icons and labels make it clear what each tool does
US-10
Page Editor — Responsive Preview
As a merchant, I want to see how my page looks on mobile so I can ensure it works on all devices.
Acceptance Criteria
  • Mobile toggle button switches preview to 375px width
  • Page content reflows responsively for mobile viewport
  • Device toggle buttons show clear active state (desktop/mobile)
  • Mobile preview accurately represents how the page appears on phones
  • Text, images, and buttons remain readable and functional on mobile
  • Toggle between desktop and mobile views is instantaneous
US-11
Subscription & Publish
As a merchant, I want to choose a plan and publish my page so it goes live.
Acceptance Criteria
  • Monthly tier ($25/mo) and Yearly tier ($20/mo) options are displayed side-by-side
  • Yearly tier shows annual savings and "Best Value" badge
  • Feature comparison list shows what's included in each plan
  • 14-day free trial is prominently mentioned
  • Plan toggle button (Monthly/Yearly) switches between billing options
  • CTA button ("Start Free Trial" or "Choose Plan") is clearly visible
  • Pricing is displayed clearly with no hidden fees mentioned
US-12
Global Progress Bar
As a merchant, I want to see how far along I am in the setup process so I know what's left before my pages are ready.
Acceptance Criteria
  • A persistent progress bar is fixed below the top navigation during the setup flow (screens 1–4)
  • Four labeled steps are displayed: Enter URL, Analyze, Brand Preview, Build Pages
  • Numbered circles mark each step with active (blue), done (green), and pending (gray) states
  • A connecting line fills with a gradient as the merchant progresses through steps
  • Percentage text (0% to 100%) is displayed on the right side of the bar
  • Progress bar updates automatically when navigating between screens
  • Progress bar disappears after "Build Pages" — the dashboard, editor, and publish screens use the standard topnav
  • Active step circle has a subtle glow ring for emphasis
US-13
A/B Testing — Experiment List
Coming Soon
As a merchant, I want to see all my A/B tests so I can track their status and results.
Acceptance Criteria
  • Table displays all A/B tests with columns: Test Name, Page, Type, Status, Last Modified
  • Search functionality allows merchants to find specific tests
  • Filter options allow sorting by status (running, paused, completed, not started)
  • Status badges are color-coded for easy visual identification
  • "Create New Experiment" CTA button is prominently displayed
  • Clicking a table row opens detailed results view for that experiment
  • Table is sortable by clicking column headers
US-14
A/B Testing — Create Experiment (Page Selection)
Coming Soon
As a merchant, I want to select which page to test so I can improve specific funnel stages.
Acceptance Criteria
  • Four page cards are displayed representing each funnel stage
  • Each card shows icon, stage name, description, and readiness score
  • Single selection mode: only one page can be selected at a time
  • Selected page shows a visual checkmark or highlight
  • Cancel button returns user to previous screen without saving
  • Continue button advances to variant setup with selected page
  • Cards are clickable to select/deselect pages
US-15
A/B Testing — Create Experiment (Variant Setup)
Coming Soon
As a merchant, I want to choose which page elements to test and see AI-suggested variants.
Acceptance Criteria
  • Grid of testable page elements is displayed with clear labels
  • Impact badges (High/Medium/Low) indicate potential conversion impact per element
  • Multi-select mode allows merchant to choose multiple elements to test
  • Visual feedback shows selected/unselected state for each element
  • AI-generated variant suggestions appear below each selected element (2-3 options)
  • Custom input option allows merchant to add custom variant text
  • Test name is auto-generated from selected elements
  • Expandable config cards provide detailed settings per element
  • Create and Cancel buttons complete or exit the setup flow
US-16
A/B Testing — Results & Promotion
Coming Soon
As a merchant, I want to see test results and promote the winner so I can improve my conversion rates.
Acceptance Criteria
  • Side-by-side variant comparison displays control vs. challenger variant
  • Live page preview shows how each variant appears
  • Stats displayed: visitors, conversions, conversion rate, vs. control comparison
  • Confidence percentage is prominently shown with color-coded banner
  • "Promote Winner" button appears only when confidence >= 95%
  • Button is disabled or grayed out when confidence is below threshold
  • Clicking "Promote Winner" replaces control with winner variant
  • Test status shows when experiment is running, paused, or completed
Non-Functional Requirements
Category Requirement
Performance Site extraction completes within 30 seconds; page loads in under 3 seconds; dashboard carousel animations smooth at 60fps
Browser Support Chrome, Safari, Firefox, and Edge (latest 2 versions)
Responsive Design Mobile-first approach; minimum supported width 375px; touch-friendly interface
Accessibility WCAG AA compliance; keyboard navigation; semantic HTML; alt text on images; sufficient color contrast
Security Session-only data storage; HTTPS only; no credential storage; CSRF protection; input validation
Data Privacy Comply with GDPR and CCPA; merchant data never shared with third parties; clear privacy policy
Out of Scope
  • A/B Testing Suite (planned for future release — user stories US-13 through US-16 are deactivated)
  • Live Shopify API integration (mocked for MVP)
  • Real site crawling and analysis (templated for MVP)
  • User authentication and saved projects (single-session flow)
  • Real-time analytics data collection and reporting
  • Multi-language support (English only for v3.0)
  • Team collaboration and shared workspaces
  • Custom domain hosting or subdomain allocation
  • Email marketing integrations
  • CRM integrations (Salesforce, HubSpot, etc.)
  • Advanced segmentation and personalization
  • White-label or reseller portal
  • Mobile app (web-responsive only)
